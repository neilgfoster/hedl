# ADR-021-beyond-single-repo-orchestration — Beyond-single-repo orchestration

## Status

Proposed — deferred to v1.x — 2026-05-28 — Phase 0

## Decision

This ADR captures two related ambitions without committing to either:

- **Cross-repo / cross-Hedl orchestration** — Hedl instances aware of
  other Hedl instances, able to orchestrate workstreams that span
  multiple repos.
- **Autonomous workstream execution** — agentic systems iterating a
  workstream through to completion under template + gate discipline,
  with branch protection + PR approval (per
  [[ADR-005-self-improvement-human-gated]]) as the only human
  checkpoint.

Both are recorded as **deferred** so they:

- Surface at every existential cycle (per
  [[ADR-013-existential-cycle-at-phase-boundaries]]) as a "do we
  want to commit yet?" question.
- Inform design decisions today (no commitments that would foreclose
  either ambition).
- Are explicitly **not** committed — no implementation work, no
  contract, no migration design.

"Autonomous" never means past the merge. Branch protection + PR
review (per [[ADR-005-self-improvement-human-gated]]) remains the
human anchor; autonomy applies *within* a workstream, up to the PR.

## Context

Both ambitions are large enough to be load-bearing v1.x decisions, but
small enough today that designing them in would be premature. Recording
them as deferred ADRs serves three purposes: (a) capture the ambition
so it isn't lost, (b) surface it at every phase boundary so it doesn't
quietly accumulate without explicit commitment, (c) inform negative
design today (do nothing that forecloses either).

## Prior art

Cross-repo orchestration: monorepo task-runners (Bazel, Nx) span code
boundaries but not work-tracking. GitHub Projects (org-scope) provides
shared issue tracking but no iteration discipline. Hedl's deferred
ambition — iteration + gate spanning repos — is niche, not widely
served.

Autonomous workstream execution: agentic coding frameworks (CCPM,
Aider's batch modes, GitHub Copilot Workspace, Devin) exist; each
couples a specific orchestration model with a specific harness. Hedl's
deferred ambition is to be the **portable substrate** that any agentic
framework can drive, with deterministic gates as the trust anchor. No
direct comparator.

Neither ambition is novel by itself. The combination — portable
iteration substrate + deterministic gate + multi-repo + autonomous —
would be, if pursued.

What would have to change for Hedl to delegate: a generalised
agentic-orchestration platform that adopts Hedl's gate semantics. None
exists. (If one emerges, this ADR re-evaluates per
[[ADR-012-managed-competitive-lifecycle]].)

## Options considered

- **Drop entirely from ADR series** — rejected. Loses the explicit
  ambition; risks accumulating in tribal knowledge.
- **Commit to v0.2 implementation** — rejected. Too big, too
  speculative.
- **Record as deferred, surface at existential cycles** — chosen.

## Consequences

- ADR-021 status remains `Proposed — deferred` until and unless a
  future phase boundary decides to commit. Existential cycles ask the
  question explicitly.
- Design today must not foreclose either ambition. Concretely:
  workstream IDs must be globally unique enough to survive cross-repo
  references (UUIDs or composite keys); workstream state
  serialisation must be transport-friendly (per
  [[ADR-022-pm-system-pluggability]]); deterministic gate exit codes
  must remain machine-consumable.
- No backlog items generated by this ADR. Implementation is not
  committed.
